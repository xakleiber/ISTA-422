 - 1. What is an App Service plan?
	- An App Service plan defines the capacity and resources to be shared among one or more app services that are assigned to that plan.

 - 2. What is a Web App?
	- A Web App is a web application that is hosted in an App Service.

 - 3. Name 3 Options for creating Web Apps.
	- Azure Marketplace, Visual Sudio Code, Visual Studio

 - 4. What is an App Service?
	- A service that hosts one of five applicaitons:
		- Web Apps
		- Mobile Apps
		- Logic Apps
		- API Apps
		- Function Apps

 - 5. What languages are supported by a web app?
	- .NET, PHP, Java, Python, and Node.JS

 - 6. What does the setting icon do? This opens a new blade called Settings.
	- Displays by default when you open the Web App blade.

 - 7. What is the purpose of swap?
	- Swaps the versions deployed to two different deployment slots. For example, if you have a production slot and a staging slot, you can publish your web app to staging and test it. When you’re satisfied with it, you can promote it to production by using the Swap option. When you’re sure everything is working okay, you can remove the staging version.

 - 8. What is Reset Publish Profile?
	- Resets the publishing credentials and invalidates the old credentials. These credentials are used for FTP and Git access.

 - 9. What are the Application Settings?
	- Values that apply to your web app.

 - 10. What is the Scale out (App Service Plan)?
	- Lets you set up autoscaling.

 - 11. What do the PGP, Jave and Python versions allow you to do?
	- Allows you to set the version to be run for the App Service. PHP 5.4, 5.5, 5.6, and 7.0 are supported. Java 7 and 8 are supported. For Python, versions 2.7 and 3.4 are supported.
